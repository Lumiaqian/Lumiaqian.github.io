最近学习了Netty的相关知识，顺便复习了一些TCP的相关知识。

TCP

​ TCP（Transmission Control Protocol 传输控制协议）是一种面向连接的、可靠的、基于字节流的传输层通信协议。

使用三次握手协议建立连接。TCP三次握手的过程如下：

  1. 客户端发送SYN(SEQ=x)报文给服务端，进入SYN_SEND状态。
  2. 服务端收到SYN报文，回应一个SYN(SEQ=y) ACK(ACK=x+1)报文，进入SYN_RECV 状态。
  3. 客户端收到服务端发送的SYN报文，回应一个ACK(ACK=y+1)报文，进入Established状态。

而TCP终止连接需要四次握手，这是因为TCP的半关闭（half-close）。过程如下：

  1. 假设客户端主动发请中断请求，也就是发送FIN报文，表示数据发送完毕。
  2. 服务端收到这个请求，但是服务端还有数据要发给客户端，则不必急着关闭Socket，这时就先发送一个ACK报文过去，这时候客户端就进入了FIN_WAIT状态，继续等待服务端发送FIN报文。
  3. 当服务端数据发完了，则向客户端发送FIN报文，客户端收到FIN报文，这时候可以关闭连接了，怕服务端不知道连接要关闭，客户端会发送一个ACK报文给服务端，客户端进入TIME_WAIT状态。
  4. 服务端收到ACK，服务端就知道可以断开连接了，客户端等待一段时间，没有收到消息说明服务端已经关闭，那么客户端就可以关闭了。
