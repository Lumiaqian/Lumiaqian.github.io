学习Netty之NIO

发表于 2018-05-20 | 分类于 Java ， Netty | 阅读次数：
字数统计: | 阅读时长 ≈

学习Netty得先了解NIO，因为Netty是NIO框架。

NIO是什么？

了解NIO之前，先看看什么是传统I/O也就是BIO，BIO是同步并阻塞，服务端实现模式是一个连接一个线程，如果请求量很大，就会出线很多线程，开销非常大。

为了解决这个问题，改良了BIO，也就有了NIO.

NIO是：同步不阻塞的I/O，服务端实现模式是一个请求一个线程，也就是客户端所有请求都会在多路复用器上注册，多路复用器轮询到连接有I/O请求时才会启动一个线程进行处理。

还有一个AIO也成为NIO.2，是异步非阻塞，服务器实现模式是一个有效的请求一个线程，也就是客户端的I/O请求都是有OS先完成了再通知服务器应用去启动线程进行处理。

NIO与BIO的区别：
  • BIO面向流，Stream是单向的，BIO的各种流是阻塞的
  • NIO面向缓冲区，Channel是双向的，是非阻塞的
适用场景

BIO:适用于连接数目比较少且固定架构，这种方式对服务器资源要求比较高，并发局限于应用中

NIO:适用于连接数目多且比较短（轻操作）的架构，比如聊天服务器，并发局限于应用中

AIO:适用于连接数目多且比较长（重操作）的架构，比如相册服务器，充分调用OS参与并发操作

基本组成

在Java中NIO是由三个主要部分组成：

  • Buffer
  • Channel
  • Selector
Buffer

缓冲区与Channel交互，数据是从Channel读入缓冲区，从缓冲区写入Channel。

buffer实际上是一个容器，一个连续数组，它通过几个变量来保存这个数据的当前位置状态：

  • capacity：容量，缓冲区能容纳元素的数量
  • position：当前位置，是缓冲区中下一次发生读取和写入操作的索引，当前位置通过大多数读写操作向前推进
  • imit：界限，是缓冲区中最后一个有效位置之后下一个位置的索引

filp方法：将position给limit，position置为0，也就是切换读写模式

clear方法：清空缓冲区，将position置为0，capacity的值给limit

rewind方法：重绕此缓冲区，将position置为0

Channel

表示I/O源与目标打开的连接，是双向的，但不能直接访问数据，只能与Buffer交互。

Selector

Selector是一个多路复用器，Selector可以使一个单独的线程管理多个Channel。简单来说，Selector会不断的轮询注册在上面的Channel，如果某个Channel上发生读或写事件，这个Channel就处于就绪状态，会被Selector轮询出来，然后通过SelectionKey可以获得就绪Channel的集合，进行后续I/O操作。
